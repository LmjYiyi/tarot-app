  1. 洗牌是观赏，不是参与（最关键）                                                        
                                                                                           
  InteractiveDeck.tsx:241,258-315 — shuffleDurationMs = 4200ms                             
  固定时长，用户只是看一段动画。                            
                                                                                           
  真人占卜里洗牌是 问者把意念注入牌堆 的环节，是整个仪式的灵魂。现在的实现把这个最重要的环 
  节变成了一段华丽的过场动画，问者完全没有"我亲手把问题揉进了这副牌里"的感受。
                                                                                           
  建议：长按 / 持续触摸 / 反复滑动来推进洗牌，"觉得可以了就松手"。Fisher–Yates             
  可以做成增量式（用户每次手势触发一次 swap，到一定阈值后允许停止）。这是 Galaxy
  Tarot、Labyrinthos 等成熟产品的做法。                                                    
                                                            
  2. FocusPicker 只有 7 种可能结果（伪随机）                                               
  
  InteractiveDeck.tsx:768-773 —                                                            
  const offsets = useMemo(() => {                           
    const step = TOTAL_CARDS / FOCUS_VISIBLE; // 78 / 7
    return Array.from({ length: FOCUS_VISIBLE }, (_, i) =>                                 
      Math.min(TOTAL_CARDS - 1, Math.floor(i * step + step / 2)),
    );                                                                                     
  }, []);                                                                                  
  7 个槽位对应固定的 7 个牌堆 index（约                                                    
  5、16、27、38、49、60、71）。无论问者「凭直觉」选哪一张，输出空间永远只有这 7            
  张牌。这是仪式上的 致命漏洞 — 真人扇牌至少给问者面对 21 张以上才有"扇"的感觉。           
                                                                                
  建议：要么把单牌仪式改成 21–25 张小扇形（牌堆里随机挑 21 个 index，不重复）；要么保留 7  
  张，但 每次进入或点 "换一组" 重抽 index。                                                
  
  3. PilePicker 的三叠是按物理顺序切分                                                     
                                                            
  InteractiveDeck.tsx:1171-1175 —                                                          
  const piles = [                                           
    { label: "左/自我", range: [0, 26] },
    { label: "中/情境", range: [26, 52] },
    { label: "右/外界", range: [52, 78] },                                                 
  ];                                      
  问者选「左/自我」就只能抽到洗牌后前 26 张里的牌 — 牌池被削减到三分之一。真实切牌是把整副 
  重新分作三堆，每堆里都可能出现任意一张牌（包括同一张大阿尔克那）。                      
                                                                                           
  建议：选定一叠后，再对该叠（或对全 78 张做"按 3           
  取模"分组）做一次内部洗牌，让每一叠都有完整 78                                           
  张的可能性，但仍受切牌位置影响。最简单的实现：选叠后，从全 78 张里以叠 index 为 seed
  重新洗，截取 26 张展示。                                                                 
                                                            
  4. NumberPicker 抽多张是连续切片

  InteractiveDeck.tsx:1380-1384 —                                                          
  const start = Math.max(0, Math.min(TOTAL_CARDS - cardCount, value - 1));
  for (let i = 0; i < cardCount; i += 1) {                                                 
    indices.push((start + i) % TOTAL_CARDS);                                               
  }                                         
  "心象数字"模式抽 5 张时，等于抽了第 N、N+1、N+2、N+3、N+4 张 —                           
  一连五张相邻牌。仪式上完全说不通（每张牌应当对应一次独立的直觉发声）。
                                                                                           
  建议：要求问者输入 cardCount
  个数字（"默念三次，每次记下浮现的第一个数字"），然后映射到牌堆。                         
                                                            
  5. 切牌的 cutAt 与"选牌"概念错位                                                         
                                                            
  shuffle.ts:117-122 的 applyCut 把数组旋转后 cutAt: cutIndex — 但 FanPicker/PilePicker 在 
  selecting 阶段使用的是 0–77 的 新顺序                     
  index。从专业角度看这没错（切牌就是移动顶端），但用户在切牌阶段看到的"第 36              
  张"和扇形里看到的位置毫无视觉对应关系。仪式语义是断裂的。 

  建议：切牌动画里把"切到的位置"用一道亮线视觉化保留到下一阶段（比如扇形里那张牌带一个"切口
  "标记）。让问者感到"我刚才切的位置确实进入了下一步"。
                                                                                           
  二、仪式/体验问题（不致命但显得不专业）                                                  
  
  6. 没有"问题先于洗牌"的强约束                                                            
                                                            
  专业占卜的第一步永远是 把问题说清楚。当前 question                                       
  字段虽然存在，但流程上不强制问者在洗牌前写下问题。建议：洗牌按钮在 question
  为空时禁用，洗牌过程中把问题悬浮显示在中央（"反复默念这一句"）。                         
                                                            
  7. 同一问题允许无成本重抽                                                                
  
  现在如果用户不喜欢结果，可以再来一次。塔罗的行业守则是 同一问题 24                       
  小时内不重复占卜（Rachel Pollack、Mary Greer 都明确写过这条）。建议：同一 question +
  spread 组合，立即重抽时弹一句温和劝阻 —                                                  
  "牌已经回答过你，再问一次它会含糊起来"。这是非常重要的仪式约束。

  8. 逆位率固定 35%，没有"关闭逆位"开关                                                    
  
  shuffle.ts:5 DEFAULT_REVERSED_RATE = 0.35。很多入门读者（包括 Biddy Tarot、TarotLore     
  等流派）建议初学者关闭逆位。建议：在 spread 选择或 settings 里给一个 "包含逆位 / 仅正位"
  的开关，默认包含逆位。
